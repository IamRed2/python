## 매직 메서드

기본 사용법에서 설명하지 않은 매직 메서드를 설명한다.

파이썬은 특정 연산이나 문법이 사용될 때 자동으로 호출되는 특수한 메서드를 제공한다. 이러한 메서드를 매직 메서드(Magic Method) 또는 던더 메서드(Dunder Method)라고 한다.

던더는 Double Under의 줄임말로, 이름 앞뒤에 언더스코어 두 개(`__`)가 붙어 있기 때문에 붙은 이름이다.

### str

print()를 사용하거나 str()로 형변환할 때 호출되는 메서드이다.

```python
class Person:
    ...
    def __str__(self):
        return f"이름: {self.name}"
```

### repr

객체를 개발자가 보기 좋은 문자열로 변환할 때 사용한다. repr() 함수가 호출될 때 사용되는 메서드이다.

```python
class Person:
    ...
    def __repr__(self):
        return f"Person(name={self.name})"
```

### len

len() 함수가 호출될 때 동작하는 메서드이다.

```python
class Box:
    def __init__(self, items: list):
        self.items = items

    def __len__(self):
        return len(self.items)
```

### getitem

객체를 리스트처럼 인덱싱할 수 있게 해주는 메서드이다.

```python
class Numbers:
    def __init__(self, data):
        self.data = data

    def __getitem__(self, index):
        return self.data[index]
```

### setitem

인덱스를 사용하여 값을 대입할 수 있게 해주는 메서드이다.

```python
class Numbers:
    def __init__(self, data):
        self.data = data

    def __setitem__(self, index, value):
        self.data[index] = value
```

### iter와 next

[이터레이터](#iterator-이터레이터)를 구현할 때 사용한다.

### enter와 exit

[컨텍스트 매니저](#context-manager-컨텍스트-매니저)에 사용된다.

### call

객체를 함수처럼 사용할 수 있게 해준다.

```python
class Adder:
    def __call__(self, a, b):
        return a + b

adder = Adder()
print(adder(1, 2))
```

출력

```text
3
```

### add, sub, 등

객체끼리 연산할 수 있게 해주는 메서드이다.

대표적으로 `__add__`(+), `__sub__`(-), `__mul__`(*), `__truediv__`(/) 등이 있다.

```python
from __future__ import annotations

class Num:
    def __init__(self, x):
        self.x = x

    def __add__(self, other: Num) -> Num:
        return Num(self.x + other.x)

    def __sub__(self, other: Num) -> Num:
        return Num(self.x - other.x)

    def __str__(self) -> str:
        return str(self.x)
    
a = Num(10)
b = Num(5)
c = a + b

print(a)
print(b)
print(c)
```

참고로 __iadd__는 += 연산자를 사용할 때 호출된다.

이 메서드는 연산 결과를 반환해야 하며, 보통 자기 자신을 수정한 뒤 self를 반환한다. 이는 내부적으로 다음과 비슷하게 동작하기 때문이다.

```python
a = a.__iadd__(b)
```

지금까지 살펴본 것처럼 파이썬의 많은 문법은 매직 메서드로 구현되어 있다.

따라서 매직 메서드를 구현하면 객체를 함수처럼 호출하거나, 리스트처럼 접근하거나, 연산자를 사용할 수 있도록 만들 수 있다.

## property

클래스의 변수를 직접 접근하지 않고, 메서드를 통해 제어된 접근을 가능하게 하는 기능이다. 메서드를 변수처럼 사용할 수 있게 해준다.

```python
class Person:
    def __init__(self, name):
        self._name = name

    @property
    def name(self):
        return self._name

    @name.setter
    def name(self, value):
        self._name = value

p = Person("신")
print(p.name)
p.name = "신이었던 것"
print(p.name)
```

출력

```text
신
신이었던 것
```

위 코드에서 값을 가져오는 부분을 게터(Getter)라고 하며 @property를 사용한다.  
값을 설정하는 부분을 세터(Setter)라고 하며 @이름.setter를 사용한다.

흥미로운 점은 실제로는 메서드임에도 다음과 같이 변수처럼 사용할 수 있다는 것이다.

```python
print(p.name)
p.name = "새 이름"
```

내부적으로는 다음과 비슷하게 동작한다.

```python
p.name()
p.name("새 이름")
```

물론 실제로 이렇게 변환되는 것은 아니지만, 사용하는 입장에서는 메서드를 변수처럼 사용할 수 있다고 생각하면 된다.

또 다른 예시로는 이것이 있다.

```python
class Person:
    def __init__(self, age):
        self._age = age

    @property
    def age(self):
        return self._age

    @age.setter
    def age(self, value):
        if value < 0:
            raise ValueError("나이는 음수가 될 수 없습니다.")

        self._age = value
```

## dataclass

클래스를 만들다 보면 `__init__`, `__repr__`, `__eq__` 등을 반복해서 작성하는 경우가 많다. dataclass는 이러한 메서드를 자동으로 생성해 주는 기능이다.

다음 코드를 보자.

```python
from dataclasses import dataclass

@dataclass
class Point:
    x: int
    y: int

p = Point(3, 5)

print(p)
```

출력

```text
Point(x=3, y=5)
```

위 코드에는 __init__이나 __repr__이 없지만 객체 생성과 출력이 가능하다.

실제로 dataclass는 내부적으로 다음과 비슷한 코드를 자동으로 만들어 준다.

```python
class Point:
    def __init__(self, x: int, y: int):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"Point(x={self.x}, y={self.y})"
```

또한 __eq__도 자동으로 생성되기 때문에 다음과 같은 비교도 가능하다.

```python
p1 = Point(1, 2)
p2 = Point(1, 2)

print(p1 == p2)
```

출력

```text
True
```

따라서 데이터를 저장하는 용도의 클래스를 만들 때 매우 편리하게 사용할 수 있다.

## Iterator (이터레이터)

이터레이터는 다음 값을 하나씩 반환하는 객체이다.

파이썬의 `for`문은 이터레이터를 이용하여 반복을 수행하며, 리스트, 문자열, 딕셔너리 등 대부분의 반복 가능한 객체도 내부적으로 이터레이터를 사용한다.

다음은 1부터 max_ 까지 반복하는 예이다.

```python
class Counter:
    def __init__(self, max_):
        self.max_ = max_
        self.current = 0

    def __iter__(self):
        return self

    def __next__(self):
        if self.current < self.max_:
            self.current += 1
            return self.current
        raise StopIteration

c = Counter(3)

for i in c:
    print(i)
```

출력

```text
1
2
3
```

이렇게 작성되는 이유는 for문이 내부적으로 다음과 비슷하게 동작하기 때문이다.

```python
it = iter(c)

while True:
    try:
        value = next(it)
        print(value)
    except StopIteration:
        break
```

즉, iter() 함수는 __iter__를 호출하고, next() 함수는 __next__를 호출한다. 따라서 Counter는 __iter__와 __next__를 구현하여 반복 가능한 객체가 된 것이다.

또한 더 이상 반환할 값이 없을 때는 StopIteration 예외를 발생시켜 반복이 끝났음을 알려야 한다.

그런데 for문을 2개를 사용하여보자.

```python
c = Counter(3)

for i in c:
    print(i)

for i in c:
    print(i)
```

출력

```text
1
2
3
```

출력이 한 번만 나온다. 그 이유는 반복이 끝난 후에도 `c.current == 3` 상태가 유지되기 때문이다.

이터레이터는 현재 위치를 기억하므로 일반적으로 재사용할 수 없다. 따라서 다시 처음부터 반복하려면 새로운 이터레이터를 생성해야 한다.

```python
class Counter:
    ...
    def __iter__(self):
        return Counter(self.max_)
```

따라서 `반복 가능한 객체(이터러블)`와 `실제로 값을 하나씩 반환하는 객체(이터레이터)`는 서로 다른 개념이다.

그리고 typing 모듈에 타입 힌트로도 이것이 있다.

- Iterable[int] : 정수를 반복해서 얻을 수 있는 객체
- Iterator[int] : 정수를 하나씩 반환하는 이터레이터

## Generator (제너레이터)

앞에서 이터레이터를 만들 때 `__iter__`와 `__next__`를 직접 구현했다. 하지만 이런 코드는 다소 길고 작성하기 번거롭다.

이를 더 간단하게 만들기 위해 제너레이터(Generator)를 사용할 수 있다.

```python
def counter(max_):
    current = 0

    while current < max_:
        current += 1
        yield current

for i in counter(3):
    print(i)
```

출력

```text
1
2
3
```

여기서 yield를 직역하면 생산하다이고 값을 반환하면서 함수의 실행 상태를 저장한다.

그리고 다음 값이 필요할 때 이전 위치부터 다시 실행된다.

```python
def test():
    print("A")
    yield 1

    print("B")
    yield 2

    print("B")
    yield 3

g = test()

print(next(g))
print(next(g))
```

출력

```text
A
1
B
2
```

첫 번째 next 호출 시 yield 1까지 실행되고 함수가 일시 중지된다.

두 번째 next 호출 시 이전 위치부터 다시 실행되어 yield 2까지 진행된다.

그리고 함수 실행 시 제너레이터 타입을 반환한다.

```python
print(type(test))
print(type(test()))
```

출력

```text
<class 'function'>
<class 'generator'>
```

이처럼 제너레이터는 이터레이터를 간단하게 만들기 위한 기능이다.

yield가 포함된 함수는 자동으로 이터레이터가 되며, __iter__와 __next__를 직접 구현하지 않아도 된다.

또한 필요한 시점에만 값을 생성하므로 메모리를 효율적으로 사용할 수 있다.

그리고 앞에서 나온 제너레이터 표현식(Generator Expression)을 사용하면 제너레이터를 간단하게 생성할 수 있다.

```python
nums = (i for i in range(5))
```

## Context Manager (컨텍스트 매니저)

파이썬에서 with문을 사용할 때 사용되는 기능이다.

```python
with open("data.txt", "r") as f:
    text = f.read()
```

위 코드에서는 파일을 열고 내용을 읽은 뒤 자동으로 파일을 닫는다.

만약 with를 사용하지 않는다면 다음과 같이 직접 닫아야 한다.

```python
f = open("data.txt", "r")

try:
    text = f.read()
finally:
    f.close()
```

이처럼 사용이 끝난 뒤 정리해야 하는 작업을 자동으로 처리하기 위해 컨텍스트 매니저를 사용한다.

```python
class Message:
    def __enter__(self):
        print("시작")
        return self

    def __exit__(self, exc_type, exc_value, traceback):
        print("종료")

with Message():
    print("작업 중")
```

출력

```text
시작
작업 중
종료
```

`__enter__`는 with 블록에 진입할 때 호출되고, `__exit__`는 with 블록을 벗어날 때 호출된다.

또한 `__exit__`는 예외가 발생하더라도 호출된다.

```python
try:
    with Message():
        print("작업 중")
        raise ValueError("에러")
except ValueError as e:
    print(e)
```

출력

```text
시작
작업 중
종료
에러
```

그리고 `__exit__`의 매개변수는 예외 정보를 전달받기 위해 사용된다.

- `exc_type` : 발생한 예외의 타입
- `exc_value` : 발생한 예외 객체
- `traceback` : 예외가 발생한 위치 정보

예외가 발생하지 않았다면 모두 `None`이 전달된다. 이것을 이용해 발생한 예외 정보를 받아 정리 작업을 수행할 수 있다.
