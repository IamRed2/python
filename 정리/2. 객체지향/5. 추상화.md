# 추상화

추상화(Abstraction)는 객체가 반드시 가져야 하는 기능만 정의하고, 구체적인 구현은 자식 클래스가 담당하도록 하는 것이다.

예를 들어 모든 동물은 소리를 낼 수 있지만, 어떤 소리를 낼지는 동물마다 다르다.

이러한 경우 부모 클래스에서는 "소리를 낸다"라는 기능만 정의하고, 실제 구현은 자식 클래스가 담당하도록 만들 수 있다.

## 사용법

abc 모듈을 사용하여 추상 클래스를 만들 수 있다. abc는 Abstract Base Class의 약자이다. abstract는 직역하면 "추상적인"이라는 뜻이다.

```python
from abc import ABC, abstractmethod

class Animal(ABC):
    @abstractmethod
    def speak(self):
        ...

class Cat(Animal):
    def speak(self):
        print("야옹")

class Dog(Animal):
    def speak(self):
        print("멍멍")
```

Animal 클래스는 speak 메서드를 반드시 구현해야 한다는 규칙만 정의한다.

따라서 Animal을 상속받는 클래스는 speak 메서드를 반드시 구현해야 하며, 구현하지 않으면 객체를 생성할 수 없다.

다음을 추가하여 실행해 보자.

```python
class Bird(Animal): ...

bird = Bird()
```

그러면 다음과 같은 에러가 발생한다.

> TypeError: Can't instantiate abstract class Bird without an implementation for abstract method 'speak'

간단히 말하면 speak 추상 메서드를 구현하지 않았기 때문에 Bird 객체를 생성할 수 없다는 의미이다.

즉, 추상 클래스는 "이 메서드는 반드시 구현해야 한다"라는 규칙을 정의하는 데 사용된다.

## 마지막

추상화는 객체가 어떤 기능을 가져야 하는지를 정의하는 역할을 한다. 이를 통해 자식 클래스들이 공통된 인터페이스를 가지게 되고, 다형성을 더욱 안전하게 사용할 수 있다.
