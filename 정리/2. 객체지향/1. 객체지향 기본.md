# 객체지향이 생긴 이유

옛날에는 데이터를 함수에 전달하여 결과를 얻거나 데이터를 변경하는 절차지향 방식으로 프로그램을 개발했다. 하지만 전역 변수의 사용으로 인해 데이터가 예상치 못하게 변경되거나, 개발자마다 프로그램을 바라보는 방식이 달라 협업에 어려움이 있었다. 또한 프로그램의 규모가 커지고 함수의 수가 증가할수록 코드의 유지보수가 어려워지는 문제도 발생했다.

이러한 문제를 해결하기 위해 관련된 데이터와 기능을 하나의 단위로 묶어 관리하는 객체지향 프로그래밍이 등장하였다. 객체지향은 프로그램을 객체들의 상호작용으로 구성하여 복잡한 소프트웨어를 보다 체계적으로 개발하고 유지보수할 수 있도록 하는 설계 방법이다.

## 객체지향의 기본 개념

1. 클래스(Class): 객체를 만들기 위한 설계도
2. 객체(Object): 클래스를 바탕으로 메모리에 생성된 실체
3. 인스턴스(instance): 특정 클래스로부터 생성된 객체

s = str() 을 하면 s는 객체이고 str 클래스의 인스턴스이다.

## 객체지향의 특징

1. 캡슐화(Encapsulation): 데이터와 기능을 하나의 객체로 묶고, 외부에서 직접 접근하지 못하도록 보호하는 것이다.
2. 상속(Inheritance): 기존 클래스의 속성과 기능을 물려받아 새로운 클래스를 만드는 것이다.
3. 다형성(Polymorphism): 같은 메서드 호출이라도 객체의 종류에 따라 다른 동작을 수행하는 것이다.
4. 추상화(Abstraction): 복잡한 내부 구현은 숨기고 필요한 기능만 제공하는 것이다.
