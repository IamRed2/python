# 캡슐화

캡슐화(Encapsulation)는 객체의 내부 데이터를 보호하고, 외부에서는 필요한 기능만 사용하도록 만드는 것이다.

예를 들어 은행 계좌를 생각해 보자.

```python
class Account:
    def __init__(self, money):
        self.money = money

account = Account(1000)

account.money = -1000000
```

위 코드는 잔액이 음수가 되는 문제가 발생한다. 따라서 보통은 데이터를 직접 수정하지 못하게 하고 메서드를 통해 접근하도록 만든다.

```python
class Account:
    def __init__(self, money):
        self._money = money

    def deposit(self, amount):
        self._money += amount

    def withdraw(self, amount):
        if self._money >= amount:
            self._money -= amount

account = Account(1000)

account.deposit(500)
account.withdraw(300)
```

여기서 `_money`처럼 앞에 언더스코어(_)를 붙인 변수는 내부 구현이라는 의미를 가진다. 그리고 객체의 상태를 메서드를 통해 관리할 수 있다.

하지만 관례적이기에 실제로 막지는 못한다. 왜냐하면 파이썬은 접근을 강제로 제한하기보다 개발자의 책임을 중요하게 생각한다.

파이썬 커뮤니티에서는 이를 "We're all consenting adults here."(우리는 모두 책임질 수 있는 성인들이다.)라는 말로 표현하기도 한다.

따라서 `_name`과 같은 관례를 통해 내부 구현임을 나타내며, 정말 필요하다면 외부에서도 접근할 수 있다.

## 언더스코어 2개로 접근을 막을 수 있는가?

파이썬 기초를 알려고 이곳저곳 찾다 보면 변수 앞에 언더스코어를 2개 붙인 것이 접근을 막을 수 있다고 한 것을 볼 수 있다. 그런데 이것은 접근을 막기 위해 사용하는 기능이 아니다.

이 기능은 이름 맹글링(Name Mangling)이며, 상속 과정에서 이름 충돌을 방지하기 위한 기능이다. 실제로 PEP 8에서도 언더스코어 두 개는 상속받는 클래스와의 이름 충돌을 피하고 싶을 때 사용하는 것을 권장하고 있다.

```python
class Animal:
    def __init__(self):
        self.__name = "Animal"

    def print_animal(self):
        print(self.__name)

class Dog(Animal):
    def __init__(self):
        super().__init__()
        self.__name = "Dog"

    def print(self):
        print(self.__name)

dog = Dog()

dog.print()
dog.print_animal()
```

출력

```python
Dog
Animal
```

부모 클래스의 `__name`과 자식 클래스의 `__name`이 서로 다른 값을 가지는 것을 알 수 있다. 이는 이름 맹글링(Name Mangling) 때문에 내부적으로 다음과 같이 이름이 변경되기 때문이다.

```python
_Animal__name
_Dog__name
```

따라서 부모 클래스와 자식 클래스에서 같은 이름의 속성을 사용하더라도 서로 충돌하지 않는다.

실제로 다음과 같이 접근할 수도 있다(IDE 에서 빨간줄을 띄우지만 실행된다).

```python
print(dog._Dog__name)
print(dog._Animal__name)
```

결과적으로는 완전한 접근 제한은 없다.

## 마지막

파이썬의 캡슐화는 접근을 강제로 제한하기보다 관례를 통해 내부 구현을 보호하는 방식으로 이루어진다. 따라서 `_name`을 사용하여 내부 구현임을 나타낸다.

중요한 것은 접근을 막는 것이 아니라 객체의 상태를 올바른 방법으로 관리하도록 만드는 것이다.
