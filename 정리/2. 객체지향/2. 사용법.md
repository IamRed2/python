## 문법

먼저 클래스 문법은 다음과 같다.

```python
class 클래스_이름:
    ...
```

예약어로 `class`를 사용하고 그 뒤에 이름, 다음에 내용물을 넣으면 된다.

그리고 객체를 만들 때는 클래스 이름 뒤에 괄호를 붙이면 된다.

```python
객체 = 클래스_이름()
```

### 생성자

객체가 생성될 때 자동으로 실행되는 메서드(함수)를 생성자라고 한다.

```python
class Point:
    def __init__(self):
        ...
```

생성자의 이름은 반드시 `__init__`이어야 한다. 그리고 생성자를 직접 만들지 않으면 부모 클래스의 생성자가 호출된다. 일반적인 경우에는 아무 일도 하지 않는 생성자가 실행된다고 생각해도 된다. 왜냐하면 실제로는 object.__init__()이 호출되기 때문이다. 이것에 대해서는 다음 장에서 나오는 상속에서 알아볼 것이다.

생성자의 매개변수를 사용하여 객체의 초기 상태를 지정할 수도 있다.

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(3, 5)
```

### 객체 생성

클래스를 정의했다면 객체를 생성할 수 있다.

```python
p1 = Point(3, 5)
p2 = Point(1, 2)
```

여기서 `p1`과 `p2`는 서로 다른 객체이며 각각 독립적인 데이터를 가진다.  
그리고 위처럼 생성된 객체는 변수에 저장하여 사용할 수 있으며, 하나의 클래스로 여러 객체를 만들 수 있다.

### 속성

속성(Attribute)은 객체가 가지는 데이터이다.

속성은 `.`(도트) 연산자를 사용하여 접근할 수 있다.

```python
print(p1.x, p1.y)  # 3 5
```

속성의 값을 변경하는 것도 가능하다.

```python
p1.x = 10
```

그런데 특별한 점은 __init__ 에서 생성하지 않은 속성을 대입연산자를 통해 값을 넣으려 한다면 새로운 속성이 생기게 된다.

```python
p1.z = 0

print(p1.z)  # 0
print(p2.z)  # AttributeError
```

p1에만 z라는 속성이 생성되었기 때문에 p2에서는 접근할 수 없다.  
그리고 객체가 가지는 속성을 인스턴스 속성(Instance Attribute)이라고 한다.

### 메서드

메서드(Method)는 클래스 내부에 정의된 함수이다.

메서드의 첫 번째 매개변수는 일반적으로 `self`를 사용한다. 변수이기에 꼭 `self`로 할 필요는 없지만 특별한 경우가 아니라면 `self`로 해두자.

```python
class Point:
    def move(self, x, y):
        self.x += x
        self.y += y
```

위처럼 메서드의 첫 번째 매개변수는 현재 객체를 받으며, 관례적으로 self라는 이름을 사용한다.

호출할 때는 객체를 사용한다.

```python
p.move(1, 2)
```

### self

`self`는 현재 메서드를 호출한 객체 자신을 가리킨다. 메서드를 호출할 때는 `self`를 직접 전달하지 않는다. 도트 연산자(.)의 왼쪽에 있는 객체가 자동으로 첫 번째 인수로 전달된다.

```python
p = Point(3, 5)
p.move(1, 2)
```

위 코드는 내부적으로 다음과 비슷하게 동작한다.

```python
Point.move(p, 1, 2)
```

### 클래스 속성

클래스 내부에 메서드 밖에서 선언한 변수는 클래스 속성이 된다.

```python
class Point:
    base_x = 0
    base_y = 0
```

클래스 속성은 모든 객체가 공유하며, 일반적으로 클래스 이름을 통해 접근한다.

```python
print(Point.base_x) # 0
```

그런데 다음처럼 사용한다면 결과가 이상하다고 생각할 것이다.

```python
p1 = Point(1, 1)
print(p1.base_x)    # 0

p1.base_x = 1
print(p1.base_x)    # 1
print(Point.base_x) # 0
```

이렇게 결과가 나오게 된다. 이유는 속성을 찾을 때에는 먼저 인스턴스 속성을 찾고 없을 경우 클래스 속성을 찾기 때문에 p1만의 base_x 가 생겨서 그렇다(그러니 클래스 속성은 클래스 이름으로 접근해주자).

### 매직 메서드

파이썬은 특정 동작을 직접 정의할 수 있는 매직 메서드를 제공한다.

매직 메서드는 일반적으로 앞과 뒤에 언더바(언더스코어)를 2개 쓴 형태이다. `__init__`도 매직 메서드중 하나이다.

그리고 `==` 연산은 `__eq__` 메서드를 사용하여 구현할 수 있다.

```python
class Point:
    def __eq__(self, other):
        if not isinstance(other, Point):
            return False

        return self.x == other.x and self.y == other.y
```

이제 객체끼리 비교할 수 있다.

```python
p1 = Point(0, 0)
p2 = Point(0, 1)
p3 = Point(0, 0)

print(p1 == p2) # False
print(p1 == p3) # True
```

other에는 비교 연산자의 오른쪽 객체가 전달된다. 이름이 반드시 other일 필요는 없지만, 비교 대상 객체를 의미하기 때문에 관례적으로 other라는 이름을 사용한다.

### 타입 힌트 시 주의할 점

만약 p1이 p2로 이동하기 위한 메서드를 만든다고 해보자. 그러면 다음처럼 될것이다.

```python
class Point:
    ...
    def move_to_point(self, point: Point):
        self.x = point.x
        self.y = point.y
```

보기에는 문제가 없어보인다. 그런데 실행을 시키면 NameError 가 날 것이다. 왜냐하면 클래스 내부에서 자기 자신의 타입을 타입 힌트로 사용하면, 클래스가 아직 완전히 정의되기 전에 이름을 참조하게 되므로 오류가 발생할 수 있다. 따라서 문자열로("Point" 처럼) 작성하거나 from __future__ import annotations를 사용해 타입 힌트의 해석을 나중으로 미뤄 해결한다.
